# R-phoneme — Полное руководство пользователя

> Руководство для исследователей и разработчиков: как тестировать нейросеть
> распознавания звука «Р», собирать данные для её дообучения и выявлять слабые места.

---

## 🖱️ Начало работы

1. Откройте приложение по ссылке: [lexbayart.github.io/r-phoneme](https://lexbayart.github.io/r-phoneme/)
2. Дождитесь загрузки модели (внизу появится «Модель готова»)
3. Выберите набор заданий: **«Правильное Р»** или **«Замена Р→Л»** (кнопки над логом)
4. Нажмите зелёную кнопку **▶ Начать**
5. Браузер запросит доступ к микрофону — разрешите
6. Произносите звуки по инструкции на карточке — индикатор отреагирует

### Интерфейс

Экран разделён на три панели:

```
┌──────────────┬──────────────────┬──────────────┐
│  ЛОГ         │    ЦЕНТР         │  НАСТРОЙКИ   │
│  инференса   │  карточка        │  навигация   │
│              │  задания          │  (набор Л)   │
│  Копировать  │  полосы          │  пресеты     │
│  Скачать     │  микрофон        │  слайдеры    │
│  Очистить    │  старт/стоп      │  нормализация│
│              │  скачать аудио    │              │
└──────────────┴──────────────────┴──────────────┘
```

- **Левая панель** — лог инференса с кнопками копирования и скачивания; над логом — переключатель наборов
- **Центральная панель** — карточка текущего задания, полосы уверенности, уровень микрофона, кнопки запуска, кнопка скачивания аудио
- **Правая панель** — навигация по карточкам (в наборе Л), пресеты, слайдеры настроек (в наборе Р)

## 📋 Два набора заданий

### Набор «Правильное Р» — аудит нейросети

6 карточек, покрывающих все варианты произношения «Р», в которых модель может ошибаться:

| Карточка | Что проверяет |
|----------|-------------|
| ✂️ **Раздельное** | Ловит ли модель короткие изолированные «Р» с паузами |
| 🔁 **Непрерывное** | Удерживает ли обнаружение при длительном звуке |
| 📦 **Порциями** | Корректно ли работает при прерывистом произношении (серии с вдохами) |
| 📈 **Нарастание** | Как модель реагирует на изменение громкости — от шёпота до максимума |
| 🔼 **Высокий тон** | Распознаёт ли «Р» в тонком / детском голосе |
| 🔽 **Низкий тон** | Распознаёт ли «Р» в грудном / басовом регистре |

При нажатии «Начать» запись запускается и пресеты автоматически переключаются
каждые 3 секунды — так за одну сессию собираются данные для всех конфигураций модели.
После прохождения всех пресетов запись останавливается и показывается следующая карточка.

### Набор «Замена Р→Л» — сбор данных о ложных срабатываниях

10 карточек, в которых вместо «Р» произносятся слоги с «Л» — те звуки,
которые ребёнок-логопат использует вместо «Р»:

**Мягкое ЛЬ:** ля, лё, лю, ли, ле (вместо РА, РО, РУ, РИ, РЕ)
**Твёрдое Л:** ла, ло, лу, лы, лэ (вместо РА, РО, РУ, РЫ, РЭ)

Каждая карточка отрабатывает три уровня громкости:
- **Тихо** — почти шёпот
- **Средне** — обычный разговорный голос
- **Громко** — уверенный, повышенный голос

Навигация по карточкам — в правой панели: список всех 10 заданий с отметками
выполненных (✓), текущего и ожидающих. Клик по любой карточке — переход к ней.

В этом наборе нейросеть не используется — фиксируется только уровень громкости.
Цель: собрать аудиозаписи «Л» вместо «Р» для добавления в датасет как негативных примеров.

## ⚙️ Пресеты (набор «Правильное Р»)

Пресеты — готовые конфигурации настроек модели. Во время записи пресеты
переключаются автоматически каждые 3 секунды.

| Пресет | Назначение |
|--------|-----------|
| **★ А: Лучший из логов** | Победитель тестов: порог 0.60, интервал 50мс, масштаб 48000, без нормализации |
| **Б: Норм + низкий порог** | Нормализация пика к 16000, порог 0.62. Для ситуаций с вариацией громкости |
| **В: Норм умеренная** | Нормализация к 20000, интервал 100мс. Менее агрессивный вариант |
| **Г: Комбо** | Порог 0.58, масштаб 48000, нормализация к 16000. Самый чувствительный |

## 🎚️ Ручные настройки (набор «Правильное Р»)

Все слайдеры работают в реальном времени — можно менять их, не останавливая запись.

### Порог обнаружения «Р»
- **Диапазон:** 0.50 – 0.95
- **Значение:** минимальная уверенность модели, при которой считается, что «Р» обнаружен
- **Меньше** — чувствительнее, но больше ложных срабатываний
- **Больше** — строже, но может пропускать тихие «Р»

### Масштаб аудио
- **Диапазон:** 8000 – 65536
- Коэффициент преобразования сигнала микрофона в формат модели
- Стандарт: 32768. Если модель не «слышит» тихую речь — увеличьте до 48000

### Интервал инференса
- **Диапазон:** 50 – 500 мс
- Как часто запускается анализ. 50мс = максимальная чувствительность

### Нормализация
- **Чекбокс** — делит сигнал на пиковую амплитуду и умножает на целевое значение
- **Мин. пик** — ниже этого значения нормализация не применяется (защита от тишины)
- **Целевой пик** — к какой амплитуде приводить сигнал

## 📋 Лог и экспорт данных

### Что показывает каждая строка лога

```
0012.34  r=72% n=18% u=10% pk=15234 ✓
```

| Поле | Значение |
|------|---------|
| `0012.34` | Время от начала сессии в секундах (синхронизировано с WAV) |
| `r=72%` | Уверенность, что это звук «Р» |
| `n=18%` | Уверенность, что это шум |
| `u=10%` | Уверенность, что это неизвестный звук |
| `pk=15234` | Пиковая амплитуда в окне (шкала int16) |
| `✓` | Сработало обнаружение (r% ≥ порога) |

Зелёные строки — обнаружение «Р». Серые — шум/тишина.

### Кнопки экспорта

- **Копировать** — копирует лог в буфер обмена (текст с настройками)
- **Скачать** — скачивает лог как .txt файл
- **Очистить** — очищает лог и сбрасывает аудиозапись
- **⬇ Скачать аудиозапись** — скачивает накопленную запись как WAV (16-bit PCM, 16 kHz mono)

Аудиозапись накапливается across multiple Start/Stop циклов — можно делать паузы.
Таймкоды в логе точно соответствуют позициям в WAV-файле.

## 📊 Как анализировать собранные данные

1. **Сопоставьте лог и аудио:** откройте WAV, перемотайте к таймкоду из лога — услышите, что именно было сказано
2. **Ищите пропуски:** где человек произнёс «Р», но r% низкий и ✓ не сработал → модель не распознаёт этот вариант
3. **Ищите ложные срабатывания:** где ✓ сработал, но по аудио это не «Р» → модель путает
4. **Сравнивайте режимы:** в каком из 6 заданий набор «Правильное Р» модель работает хуже?
5. **Собирайте проблемные примеры:** вырежьте из WAV фрагменты, где модель ошиблась → добавьте в датасет Edge Impulse → переобучите

---

## 🔄 Полный цикл дообучения

1. Запустите приложение, выберите набор заданий
2. Пройдите все карточки — произносите звуки по инструкциям
3. Скачайте лог (.txt) и аудиозапись (.wav)
4. Проанализируйте: в каких режимах модель ошибается
5. Вырежьте проблемные фрагменты из WAV
6. Добавьте их в датасет Edge Impulse с правильными метками
7. Переобучите модель
8. Скомпилируйте новый WASM → замените файлы в репозитории
9. Протестируйте снова — сравните результаты до и после

## ⚠️ Известные ограничения

- Модель работает в браузерах на движке Chromium (Chrome, Edge, Яндекс.Браузер). Firefox может не поддерживать WASM-модуль Edge Impulse.
- Для работы необходим доступ к микрофону. Приложение не записывает и не отправляет аудио — всё обрабатывается и сохраняется локально.
- Модель классифицирует 1-секундное окно. Короткие «Р» (< 1 сек) будут содержать тишину в окне — это влияет на уверенность.
- Нормализация пика и RMS-нормализация отключены по умолчанию — они ухудшили обнаружение в тестах.
- Модель не различает «Р» в начале, середине и конце слова — она реагирует на акустический паттерн в целом.
