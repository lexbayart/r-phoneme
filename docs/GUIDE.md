# R-phoneme — Полное руководство пользователя

> Руководство для исследователей и разработчиков: как тестировать нейросеть
> распознавания звука «Р», собирать данные для её дообучения и выявлять слабые места.

---

## 🖱️ Начало работы

1. Откройте приложение по ссылке: [lexbayart.github.io/r-phoneme](https://lexbayart.github.io/r-phoneme/)
2. Дождитесь загрузки модели (внизу появится «Модель готова»)
3. Нажмите зелёную кнопку **▶ Начать слушать**
4. Браузер запросит доступ к микрофону — разрешите
5. Произнесите звук «Р» — индикатор отреагирует

### Интерфейс

Экран разделён на три панели:

```
┌──────────────┬──────────────────┬──────────────┐
│  ЛОГ         │    ЦЕНТР         │  НАСТРОЙКИ   │
│  инференса   │  индикатор       │  режимы      │
│              │  полосы          │  пресеты     │
│  Копировать  │  микрофон        │  слайдеры    │
│  Очистить    │  старт/стоп      │  нормализация│
└──────────────┴──────────────────┴──────────────┘
```

- **Левая панель** — лог инференса: каждая строка — один анализ звука с временной меткой
- **Центральная панель** — визуальный индикатор, полосы уверенности, уровень микрофона, кнопки запуска
- **Правая панель** — режимы, пресеты, слайдеры настроек

## 🎙️ Режимы произношения

В правой панели два режима — переключайте их до или во время тестирования:

### 🔁 Непрерывное «р-р-р-р-р»
Режим для отработки артикуляционного навыка. Ребёнок произносит «Р» непрерывно,
как мотор. Цель — проверить, удерживает ли модель обнаружение при длительном звуке.

### ✂️ Раздельное «р» · «р» · «р»
Режим для проверки закрепления. Ребёнок произносит отдельные «Р» с паузами.
Цель — проверить, ловит ли модель кратковременные звуки.

При переключении в лог попадает цветная метка, чтобы потом анализировать результаты
по режимам отдельно.

## ⚙️ Пресеты

Пресеты — это готовые конфигурации настроек. Каждый пресет автоматически выставляет
все слайдеры. Выберите пресет, протестируйте, скопируйте лог.

| Пресет | Назначение |
|--------|-----------|
| **★ А: Лучший из логов** | Победитель предыдущих тестов: порог 0.60, интервал 50мс, масштаб 48000, без нормализации. 30 срабатываний в тестовом логе. Рекомендован для начала. |
| **Б: Норм + низкий порог** | Нормализация пика к 16000, порог 0.62. Для ситуаций, где громкость речи сильно варьируется. |
| **В: Норм умеренная** | Нормализация к 20000, интервал 100мс. Менее агрессивный вариант — меньше ложных срабатываний. |
| **Г: Комбо** | Объединяет лучшее из А и Б: порог 0.58, масштаб 48000, нормализация к 16000. Самый чувствительный вариант. |
| **Д: Порог 0.55 без нормализации** | Экстремально низкий порог. Для проверки: не появятся ли ложные срабатывания. |

**Как пользоваться:**
1. Нажмите на пресет — слайдеры встанут автоматически
2. Начните слушать и произносите «Р» в нужном режиме
3. Скопируйте лог кнопкой «Копировать»
4. Сравните результаты разных пресетов

## 🎚️ Ручные настройки

Все слайдеры работают в реальном времени — можно менять их, не останавливая запись.

### Порог обнаружения «Р»
- **Диапазон:** 0.50 – 0.95
- **Значение:** минимальная уверенность модели (в процентах), при которой считается, что «Р» обнаружен
- **Меньше** (0.50–0.60) — чувствительнее, но больше ложных срабатываний
- **Больше** (0.80–0.95) — строже, но может пропускать тихие «Р»
- **Рекомендация:** начните с 0.60

### Масштаб аудио
- **Диапазон:** 8000 – 65536
- **Значение:** коэффициент преобразования сигнала микрофона в формат, который понимает модель
- Стандартное значение PCM 16-bit: 32768
- Если модель не «слышит» тихую речь — увеличьте до 48000
- Если слишком много ложных срабатываний в тишине — уменьшите

### Интервал инференса
- **Диапазон:** 50 – 500 мс
- **Значение:** как часто запускается анализ (каждые N миллисекунд)
- 50мс = 20 анализов в секунду — максимальная чувствительность
- 500мс = 2 анализа в секунду — экономия ресурсов
- **Рекомендация:** 50мс для точных замеров, 100мс для обычного использования

### Нормализация
- **Чекбокс «Включить нормализацию»** — делит сигнал на пиковую амплитуду и умножает на целевое значение
- **Мин. пик:** ниже этого значения нормализация не применяется (защита от тишины)
- **Целевой пик:** к какой амплитуде приводить сигнал
- **Важно:** нормализация может как улучшить, так и ухудшить обнаружение — тестируйте с включённой и выключенной

## 📋 Работа с логом

### Что показывает каждая строка

```
02:34.1  r=72% n=18% u=10% pk=15234 ✓
```

| Поле | Значение |
|------|---------|
| `02:34.1` | Время (минуты:секунды.десятые) |
| `r=72%` | Уверенность, что это звук «Р» |
| `n=18%` | Уверенность, что это шум |
| `u=10%` | Уверенность, что это неизвестный звук |
| `pk=15234` | Пиковая амплитуда в окне (шкала int16) |
| `✓` | Сработало обнаружение (r% ≥ порога) |

Зелёные строки — обнаружение «Р». Серые — шум/тишина.

### Копирование лога
Кнопка **«Копировать»** копирует в буфер обмена полный лог с заголовком настроек.
Формат удобен для вставки в чат, документ или таблицу для анализа.

### Копирование настроек
Кнопка **«📋 Копировать настройки»** в правой панели копирует текущие значения слайдеров
в текстовом виде. Удобно для передачи конфигурации коллеге.

## 📊 Как анализировать лог

1. **Посчитайте количество ✓** — сколько раз модель обнаружила «Р» за сессию
2. **Посмотрите на r% без ✓** — если r=55% при пороге 0.60, значит нужно снизить порог
3. **Сравните pk в тишине и при речи:**
   - Речь: pk обычно 2000–20000
   - Тишина: pk < 500
4. **Обратите внимание на ложные срабатывания** — строки с ✓, когда ребёнок не произносил «Р»
5. **Ищите паттерны** — в каких слогах или позициях модель ошибается

## ⚠️ Известные ограничения

- Модель работает только в браузерах на движке Chromium (Chrome, Edge, Яндекс.Браузер). Firefox может не поддерживать WASM-модуль Edge Impulse.
- Для работы необходим доступ к микрофону. Приложение не записывает и не отправляет аудио — всё обрабатывается локально.
- Модель классифицирует 1-секундное окно. Если «Р» произнесён короче 1 секунды, часть окна будет тишиной — это влияет на уверенность.
- Нормализация пика (TARGET=24000) и RMS-нормализация были отключены по умолчанию — они ухудшили обнаружение в тестах.
- Модель не различает «Р» в начале, середине и конце слова — она реагирует на акустический паттерн в целом.

## 🔄 Сбор данных для дообучения модели

Основной сценарий использования R-phoneme — **аудит нейросети**:

1. Выберите пресет (начните с ★ А)
2. Переключите режим (непрерывное или раздельное)
3. Произнесите «Р» в разных вариантах: тихо, громко, в слогах, после паузы
4. Скопируйте лог
5. Проанализируйте:
   - Где r% высокий, но ✓ не сработал → модель не уверена → нужно добавить примеры в датасет
   - Где ✓ сработал ложно → модель путает «Р» с другим звуком → нужно добавить негативные примеры
   - Где pk низкий при речи → модель не «слышит» тихое произношение → нужно добавить тихие примеры
6. Скорректируйте настройки или соберите проблемные примеры для дообучения модели

Данные из лога можно использовать для:
- Добавления новых примеров в датасет Edge Impulse
- Переобучения модели с учётом выявленных слабых мест
- Сравнения качества модели до и после переобучения
